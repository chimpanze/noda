Findings:
- **usability** (boot): Project fails to boot: services.auth.config is missing the required "database" field, and no MCP tool surfaces this requirement.
- **usability** (config-validation): mcp__noda__noda_validate_config reports "valid": true for a project that cannot boot.
- **usability** (GET /admin/stats): workflows/admin_stats.json's db.count node queries a table named "users", but the auth plugin's own get_user node hardcodes table "auth_users" — the two auth patterns (built-in auth.* nodes vs. hand-rolled JWT+users-table pattern) were mixed inconsistently in this project and are not table-compatibl
- **usability** (migrations): Nothing in migrations/ creates the tables the auth-related workflows need (neither "users" nor "auth_users"/"auth_sessions"/"auth_tokens").
